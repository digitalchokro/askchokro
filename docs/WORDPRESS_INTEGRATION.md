# WordPress Integration Plan

AskChokro provides a powerful AI Data Engine, but WordPress (which runs on PHP) cannot natively host a Node.js engine. To bring "Ask your data" to WordPress and WooCommerce users via a zero-code plugin, we are executing a phased architectural rollout.

## Phase 1: The AskChokro Microservice
Since WordPress cannot run AskChokro natively, we must provide a simple way for WordPress admins to host the engine.

**Objective:** Package AskChokro into a standalone, pre-configured Express server.
- Deliverable: A public Docker image (`digitalchokro/askchokro-microservice`).
- Architecture: This microservice will expose the standard `/api/ask` endpoint and connect directly to the WordPress MySQL database using the `@digitalchokro/db-mysql` adapter (to be developed).

## Phase 2: The WordPress Plugin (PHP & React)
We will build the actual WordPress `.zip` plugin that users install.

**Objective:** Provide a seamless UI for WooCommerce merchants.
- **Admin Settings Page (PHP):** A settings panel in `wp-admin` where the merchant enters the URL of their AskChokro microservice (from Phase 1).
- **Frontend UI (React):** A Gutenberg Block and Elementor Widget containing the AskChokro chat interface.
- **Authentication:** The plugin will generate a secure JWT signed by the WordPress secret key, proving to the microservice that the requests are coming from an authenticated WordPress admin.

## Phase 3: Tenant Isolation & Multi-Vendor
For advanced multi-vendor marketplaces (like Dokan or WCFM), vendors should only be able to query their own products and sales.

**Objective:** Integrate AskChokro's AST Scope Rewriter with WordPress user roles.
- The React UI will pass the current logged-in `vendor_id`.
- The WP Plugin will securely sign this `vendor_id` into the JWT.
- The Node.js microservice will verify the JWT, extract the `vendor_id`, and use the `tenantScoping` feature to inject `WHERE vendor_id = X` into the AST of all queries generated by the AI.
